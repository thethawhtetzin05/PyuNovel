# PyuNovel Project Report
*Date: March 18, 2026*

This report summarizes the recent core improvements made to the **PyuNovel** ecosystem (Next.js Web, Writer App, and Telegram Bot) and provides a strategic roadmap for future enhancements.

## 🛠 Recent Fixes & Engineering Improvements

### 1. Writer App (React Native/Expo)
*   **Persistent Editor State:** Fixed a critical bug where "new" chapters would regenerate local IDs on every auto-save, causing database duplication. Implemented initial draft loading from SQLite.
*   **Smart Sync Queue:** Refactored `addToSyncQueue` to update existing pending records instead of creating duplicates. Added background sync triggers to `RootLayout` and the Editor.
*   **Authentication Flow:** Implemented a robust `index.tsx` redirect logic that checks for `SecureStore` tokens, ensuring users are directed to the correct screen (Login vs. Dashboard) on startup.
*   **UI/Theming:** Standardized HSL color variables in `global.css` and `tailwind.config.js` to ensure Shadcn-like utility classes work correctly in the Native environment.

### 2. Telegram Bot Integration
*   **Database Type Safety:** Fixed the "Database Type Mismatch" error by forcing `chatId` to a string before comparing it with the D1/SQLite `TEXT` columns.
*   **Edge Context Stability:** Refactored `getServerContext` to handle optional `Better-Auth` initialization. This prevents the bot from crashing when receiving non-browser headers from Telegram.
*   **Robust Content Parser:** Optimized the `parseBulkText` regex to correctly identify chapter titles in both English and Myanmar digits (၀-၉), and verified the logic with unit tests.
*   **Enhanced UX:** Added "Processing..." status messages for long-running database operations and improved error feedback for invalid verification codes.

### 3. Core API & Infrastructure
*   **Reliable View Tracking:** Refactored `/api/novel/view` to use the improved `getServerContext` and `sql` tagged templates, ensuring stable incrementing on Cloudflare Edge.
*   **Global Context Helper:** Standardized how `DB`, `Auth`, and `Env` are accessed across all Server Actions and API Routes to prevent "Binding Missing" errors.

## 🚀 Strategic Roadmap / Next Steps

### 📈 Phase 1: Robustness & Data Integrity
*   **Conflict Resolution:** Implement "Last Write Wins" or basic diffing in the `writer-app` sync logic to prevent data loss during multi-device editing.
*   **Exponential Backoff:** Improve the sync retry logic to handle server downtime gracefully without draining user battery.
*   **Centralized Validation:** Move Zod schemas and TypeScript interfaces into a shared package to ensure 1:1 validation parity between the Web and App.

### 🎭 Phase 2: User Experience & Engagement
*   **Bot Notifications:** Expand the Telegram Bot to notify writers of new reviews, gifts, and system announcements.
*   **Enhanced Reader View:** Add font size, background, and spacing customization to the web reader for better long-form accessibility.
*   **Bulk Import in App:** Port the Telegram bulk-parsing logic to the Writer App to allow local `.txt` file imports.

### 🛡 Phase 3: Scaling & Observability
*   **Error Tracking:** Integrate **Sentry** or **Logflare** to catch silent Edge/D1 failures in production.
*   **Rate Limiting:** Implement Cloudflare Rate Limiting on public API endpoints (`/api/novel/view`, Telegram Webhook) to prevent abuse.
*   **E2E Testing:** Implement **Playwright** tests for the critical "Write-to-Read" user journey.
